# Git Conventions

## Commit Messages
- Format: `type(scope): description`
- Types: feat, fix, docs, style, refactor, perf, test, chore

## Branch Naming
- `feat/<ticket>-<short-desc>`
- `fix/<ticket>-<short-desc>`
- `chore/<desc>`

## Pull Requests
- Keep PRs small and focused.
- Include a summary and test plan.
